DDEESSCCRRIIPPTTIIOONN
       This command output the message digest of a supplied file or files in
       hexadecimal, and also generates and verifies digital signatures using
       message digests.

       The generic name, ooppeennssssll ddggsstt, may be used with an option specifying
       the algorithm to be used.  The default digest is sshhaa225566.  A supported
       _d_i_g_e_s_t name may also be used as the sub-command name.  To see the list
       of supported algorithms, use "openssl list -digest-commands"

OOPPTTIIOONNSS
       --hheellpp
           Print out a usage message.

       --_dd_ii_gg_ee_ss_tt
           Specifies name of a supported digest to be used. To see the list of
           supported digests, use the command "list --digest-commands".

       --cc  Print out the digest in two digit groups separated by colons, only
           relevant if the --hheexx option is given as well.

       --dd, --ddeebbuugg
           Print out BIO debugging information.

       --lliisstt
           Prints out a list of supported message digests.

       --hheexx
           Digest is to be output as a hex dump. This is the default case for
           a "normal" digest as opposed to a digital signature.  See NOTES
           below for digital signatures using --hheexx.

       --bbiinnaarryy
           Output the digest or signature in binary form.

       --xxoofflleenn _l_e_n_g_t_h
           Set the output length for XOF algorithms, such as sshhaakkee112288.

       --rr  Output the digest in the "coreutils" format, including newlines.
           Used by programs like sshhaa11ssuumm(1).

       --oouutt _f_i_l_e_n_a_m_e
           Filename to output to, or standard output by default.

       --ssiiggnn _f_i_l_e_n_a_m_e|_u_r_i
           Digitally sign the digest using the given private key. Note this
           option does not support Ed25519 or Ed448 private keys. Use the
           ooppeennssssll--ppkkeeyyuuttll(1) command instead for this.

       --kkeeyyffoorrmm DDEERR|PPEEMM|PP1122|EENNGGIINNEE

           ooppeennssssll--ffoorrmmaatt--ooppttiioonnss(1) for details.

       --ssiiggoopptt _n_m:_v
           Pass options to the signature algorithm during sign or verify
           operations.  Names and values of these options are algorithm-
           specific.

       --ppaassssiinn _a_r_g

       --mmaacc _a_l_g
           Create MAC (keyed Message Authentication Code). The most popular
           MAC algorithm is HMAC (hash-based MAC), but there are other MAC
           algorithms which are not based on hash, for instance ggoosstt--mmaacc
           algorithm, supported by the ggoosstt engine. MAC keys and other options
           should be set via --mmaaccoopptt parameter.

           The ooppeennssssll--mmaacc(1) command should be preferred to using this
           command line option.

       --mmaaccoopptt _n_m:_v
           Passes options to MAC algorithm, specified by --mmaacc key.  Following
           options are supported by both by HHMMAACC and ggoosstt--mmaacc:

           kkeeyy:_s_t_r_i_n_g
               Specifies MAC key as alphanumeric string (use if key contain
               printable characters only). String length must conform to any
               restrictions of the MAC algorithm for example exactly 32 chars
               for gost-mac.

           hheexxkkeeyy:_s_t_r_i_n_g
               Specifies MAC key in hexadecimal form (two hex digits per
               byte).  Key length must conform to any restrictions of the MAC
               algorithm for example exactly 32 chars for gost-mac.

           The ooppeennssssll--mmaacc(1) command should be preferred to using this
           command line option.

       --ffiippss--ffiinnggeerrpprriinntt

EEXXAAMMPPLLEESS
       To create a hex-encoded message digest of a file:
        openssl dgst -md5 -hex file.txt

       To sign a file using SHA-256 with binary file output:
        openssl dgst -sha256 -sign privatekey.pem -out signature.sign file.txt

       To verify a signature:
        openssl dgst -sha256 -verify publickey.pem \
        -signature signature.sign \
        file.txt

NNOOTTEESS
       The digest mechanisms that are available will depend on the options
       used when building OpenSSL.  The "openssl list -digest-commands"
       command can be used to list them.

       New or agile applications should use probably use SHA-256. Other
       digests, particularly SHA-1 and MD5, are still widely used for
       interoperating with existing formats and protocols.

       When signing a file, this command will automatically determine the
       algorithm (RSA, ECC, etc) to use for signing based on the private key's
       ASN.1 info.  When verifying signatures, it only handles the RSA, DSA,
       or ECDSA signature itself, not the related data to identify the signer
       and algorithm used in formats such as x.509, CMS, and S/MIME.

       A source of random numbers is required for certain signing algorithms,
       in particular ECDSA and DSA.

       The signing and verify options should only be used if a single file is
       being signed or verified.

       Hex signatures cannot be verified using ooppeennssssll.  Instead, use "xxd -r"
       or similar program to transform the hex signature into a binary
       signature prior to verification.

       The ooppeennssssll--mmaacc(1) command is preferred over the --hhmmaacc, --mmaacc and
       --mmaaccoopptt command line options.

HHIISSTTOORRYY
       The default digest was changed from MD5 to SHA256 in OpenSSL 1.1.0.
       The FIPS-related options were removed in OpenSSL 1.1.0.

       The --eennggiinnee and --eennggiinnee__iimmppll options were deprecated in OpenSSL 3.0.
