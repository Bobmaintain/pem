       openssl-speed, speed - test library performance

       ooppeennssssll ssppeeeedd [--hheellpp] [--eennggiinnee iidd] [--eellaappsseedd] [--eevvpp aallggoo] [--ddeeccrryypptt]
       [--rraanndd ffiillee......]  [--wwrriitteerraanndd ffiillee] [--pprriimmeess nnuumm] [--sseeccoonnddss nnuumm] [--bbyytteess
       nnuumm] [aallggoorriitthhmm......]

DDEESSCCRRIIPPTTIIOONN
       This command is used to test the performance of cryptographic
       algorithms.  To see the list of supported algorithms, use the _l_i_s_t
       _-_-_d_i_g_e_s_t_-_c_o_m_m_a_n_d_s or _l_i_s_t _-_-_c_i_p_h_e_r_-_c_o_m_m_a_n_d_s command. The global CSPRNG
       is denoted by the _r_a_n_d algorithm name.

OOPPTTIIOONNSS
       --hheellpp
           Print out a usage message.

       --eennggiinnee iidd
           Specifying an engine (by its unique iidd string) will cause ssppeeeedd to
           attempt to obtain a functional reference to the specified engine,
           thus initialising it if needed. The engine will then be set as the
           default for all available algorithms.

       --eellaappsseedd
           When calculating operations- or bytes-per-second, use wall-clock
           time instead of CPU user time as divisor. It can be useful when
           testing speed of hardware engines.

       --eevvpp aallggoo
           Use the specified cipher or message digest algorithm via the EVP
           interface.  If aallggoo is an AEAD cipher, then you can pass <-aead> to
           benchmark a TLS-like sequence. And if aallggoo is a multi-buffer
           capable cipher, e.g.  aes-128-cbc-hmac-sha1, then --mmbb will time
           multi-buffer operation.

       --ddeeccrryypptt
           Time the decryption instead of encryption. Affects only the EVP
           testing.

       --rraanndd ffiillee......
           A file or files containing random data used to seed the random
           number generator.  Multiple files can be specified separated by an
           OS-dependent character.  The separator is ;; for MS-Windows, ,, for
           OpenVMS, and :: for all others.

       [--wwrriitteerraanndd ffiillee]
           Writes random data to the specified _f_i_l_e upon exit.  This can be
           used with a subsequent --rraanndd flag.

       --pprriimmeess nnuumm
           Generate a nnuumm-prime RSA key and use it to run the benchmarks. This
           option is only effective if RSA algorithm is specified to test.

       --sseeccoonnddss nnuumm
           Run benchmarks for nnuumm seconds.

       --bbyytteess nnuumm
           Run benchmarks on nnuumm-byte buffers. Affects ciphers, digests and
           the CSPRNG.

       [[zzeerroo oorr mmoorree tteesstt aallggoorriitthhmmss]]
           If any options are given, ssppeeeedd tests those algorithms, otherwise a
           pre-compiled grand selection is tested.
