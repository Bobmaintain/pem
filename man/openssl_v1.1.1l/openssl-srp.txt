       openssl-srp, srp - maintain SRP password file

SSYYNNOOPPSSIISS
       ooppeennssssll ssrrpp [--hheellpp] [--vveerrbboossee] [--aadddd] [--mmooddiiffyy] [--ddeelleettee] [--lliisstt]
       [--nnaammee sseeccttiioonn] [--ccoonnffiigg ffiillee] [--ssrrppvvffiillee ffiillee] [--ggnn iiddeennttiiffiieerr]
       [--uusseerriinnffoo tteexxtt......]  [--ppaassssiinn aarrgg] [--ppaassssoouutt aarrgg] [_u_s_e_r_._._.]

DDEESSCCRRIIPPTTIIOONN
       The ssrrpp command is user to maintain an SRP (secure remote password)
       file.  At most one of the --aadddd, --mmooddiiffyy, --ddeelleettee, and --lliisstt options can
       be specified.  These options take zero or more usernames as parameters
       and perform the appropriate operation on the SRP file.  For --lliisstt, if
       no uusseerr is given then all users are displayed.

       The configuration file to use, and the section within the file, can be
       specified with the --ccoonnffiigg and --nnaammee flags, respectively.  If the
       config file is not specified, the --ssrrppvvffiillee can be used to just specify
       the file to operate on.

       The --uusseerriinnffoo option specifies additional information to add when
       adding or modifying a user.

       The --ggnn flag specifies the gg and NN values, using one of the strengths
       defined in IETF RFC 5054.

       The --ppaassssiinn and --ppaassssoouutt arguments are parsed as described in the
       ooppeennssssll(1) command.

OOPPTTIIOONNSS
       [--hheellpp]
           Display an option summary.

       [--vveerrbboossee]
           Generate verbose output while processing.
